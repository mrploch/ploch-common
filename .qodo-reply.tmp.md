#### #1 — Why "missing `style=flat`" is a false positive

shields.io's **default style is already `flat`**. From [shields.io's documentation](https://shields.io/badges/static-badge), the `style` parameter is optional, and when omitted the rendered badge is the flat style — same as setting `style=flat` explicitly.

None of the seven pre-existing badges in `README.md` set `style=flat` either:

```markdown
[![NuGet](https://img.shields.io/nuget/v/Ploch.Common.svg)](...)
[![Quality Gate Status](https://sonarcloud.io/api/project_badges/measure?project=mrploch_ploch-common&metric=alert_status)](...)
```

So the new Buy Me a Coffee badge follows the **exact same pattern** as every existing badge in the row — and renders in the same flat style. Adding `style=flat` would make the new badge more explicit than the existing ones (one explicit + seven implicit), not more consistent. The PR description's note that the badge is "flat style, not `for-the-badge`" describes the visual result, which is correct: the badge renders flat by default. Leaving as-is.
